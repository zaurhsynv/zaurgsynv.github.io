Əməliyyat Sistemləri Təhlükəsizliyi

Dərinləşmiş Təhlil və Müdafiə Strategiyaları

1. Giriş: Əməliyyat Sistemlərinin Təhlükəsizliyinin Rolu

Əməliyyat sistemləri (ƏS) bir kompüter sisteminin onurğa sütunu sayılır. Bu sistemlər istifadəçi ilə aparat vasitələri arasında körpü rolu oynayır və proqramların icrasını, resursların bölüşdürülməsini və təhlükəsizlik nəzarətini həyata keçirir. Əgər ƏS-in təhlükəsizliyi pozularsa, həmin cihazda çalışan hər bir proqram və məlumat risq altına düşür. Bu səbəbdən ƏS təhlükəsizliyi təşkilatın ümumi kiber təhlükəsizlik strategiyasının əsas sütunlarından biri olmalıdır.

2. ƏS Təhlükəsizlik Arxitekturası

2.1. Kernel səviyyəsində təhlükəsizlik

Əməliyyat sisteminin nüvəsi – kernel – bütün resurslara girişin əsas nəzarət nöqtəsidir. Kernel səviyyəsində təhlükəsizlik aşağıdakı əsas prinsiplərlə təmin edilir:

  • Process Isolation (Proseslərin İzlolyasiyası)
  • Memory Protection (Yaddaş Qorunması)
  • System Call Filtering (Syscall nəzarəti)
  • Least Privilege Access (Ən Aşağı İcazə prinsipi)

2.2. Userland səviyyəsində təhlükəsizlik

Userland – istifadəçinin birbaşa işlədiyi qatdır. Bu qatdakı təhlükəsizlik tədbirləri:

  • User Account Control (UAC) və RBAC
  • Security Policy Enforcement (Güclü siyasətlərin tətbiqi)
  • Application Whitelisting və Blacklisting

3. ƏS Təhlə Modelləri və Hücum Vektorları

3.1. Əməliyyat sisteminə qarşı əsas hücum növləri

Hücum Növü Təsviri
Privilege Escalation İstifadəçinin icazəsini artıraraq sistem səviyyəsində nəzarət əldə etməsi
Buffer Overflow Yaddaş sahələrinə zərərli kod yükləmə yolu ilə nəzarəti ələ keçirmə
Rootkit Sistemə dərinlikdə yerləşdirilən və görünməz qalan zərərli proqram
Kernel Exploit ƏS-nin nüvəsindəki zəiflikdən istifadə edərək sistemin tam nəzarətini alma
DLL Injection / Library Hijacking Sistem kitabxanalarının dəyişdirilməsi ilə zərərli kodun yeridilməsi

4. ƏS Təhlükəsizlik Mexanizmləri və Texnologiyalar

4.1. Mandatory Access Control (MAC) modelləri

  • SELinux (Security-Enhanced Linux): Qaydalara əsaslanan icazə modeli
  • AppArmor: Profil əsaslı tətbiq nəzarəti
  • Windows Integrity Control: Objektdən istifadə hüquqlarını təyin edir

4.2. Antivirus və Endpoint Detection & Response (EDR)

EDR sistemləri yalnız virus aşkarlamır, həm də davranış təhlili apararaq şübhəli fəaliyyətləri müəyyən edir.

  • BitDefender
  • CrowdStrike
  • Microsoft Defender for Endpoint

4.3. Sandboxing və Virtualization

Zərərli fəaliyyətlərə qarşı tətbiqləri izolyasiya etmək üçün istifadə olunur. Məsələn:

  • Windows Sandbox
  • Docker konteynerləri
  • Qubes OS (Security through Isolation)

5. OS-lər üzrə spesifik təhlükəsizlik xüsusiyyətləri

5.1. Windows Təhlükəsizlik Xüsusiyyətləri

  • Group Policy Objects (GPO)
  • Credential Guard və Device Guard
  • NTFS Permission Set
  • Windows Defender Exploit Guard

5.2. Linux Təhlükəsizlik Xüsusiyyətləri

  • iptables və nftables
  • SELinux/AppArmor
  • Auditd log sistemi
  • Kernel security modules (LSM)

5.3. macOS Təhlükəsizlik Arxitekturası

  • System Integrity Protection (SIP)
  • XProtect və Gatekeeper
  • Apple T2 Security Chip

6. Audit, Logging və Təhlükəsizlik Monitorinqi

6.1. Audit mexanizmləri

  • Syslog (Linux)
  • Event Viewer (Windows)
  • auditd və aureport (Linux kernel auditing)

6.2. SIEM inteqrasiyası

ƏS logları SIEM (Security Information and Event Management) sistemlərinə yönləndirilərək real-time təhlil edilir.

  • Splunk
  • Graylog
  • ELK Stack (Elastic, Logstash, Kibana)

7. ƏS Təhlükəsizliyində Yenilənmələr və Patch Management

7.1. Patch-ləmə Strategiyası

  • Automated patch deployment (WSUS, Landscape, yum-cron)
  • Critical Patch Management Cycle
  • Vulnerability scanning ilə uyğunlaşdırma (Nessus, OpenVAS)

8. Real Dünyadan Təhlükəsizlik Hadisələri

  • Stuxnet: Windows SCADA sistemlərinə yönəlmiş kernel rootkit hücumu
  • Dirty COW: Linux Kernel Privilege Escalation
  • NotPetya: Windows sistemlərini şəbəkədə lateral hərəkətlə ələ keçirmə

9. ƏS Təhlükəsizliyi üçün Sertifikatlar və Təlimlər

Sertifikat Məzmun

OSCP / OSEP ƏS zəifliklərinin istismarı və real hücum simulyasiyaları

10. Gələcək Perspektivlər

  • Zero Trust ƏS Arxitekturaları
  • AI əsaslı hücum aşkarlanması
  • Quantum-resistant cryptography
  • Edge Device OS-lərin təhlükəsizliyi (IoT, ICS)
