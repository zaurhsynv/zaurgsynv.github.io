Giriş: Active Directory nədir?

Active Directory (AD), Microsoft tərəfindən hazırlanmış bir direktor xidmətidir və əsas məqsədi şəbəkədəki resursları mərkəzləşdirilmiş şəkildə idarə etməkdir. Bu resurslara istifadəçilər, kompüterlər, printerlər, serverlər və s. daxildir. AD əsasən domen əsaslı şəbəkələr üçün nəzərdə tutulub və Windows Server əməliyyat sisteminin bir hissəsi kimi fəaliyyət göstərir.

Active Directory Strukturunun Əsas Elementləri

Domain (Domen)

AD-nin əsas struktur vahididir. Domen eyni idarəetmə siyasətlərinə tabe olan obyektlərin məcmusudur. Məsələn, example.local.

Tree (Ağac)

Birdən çox domenin bir-birinə etibar əsasında bağlandığı struktur. Eyni namespace istifadə olunur.

Forest (Meşə)

Bir və ya bir neçə ağacı birləşdirən AD strukturudur. Ən yüksək səviyyəli loqikal konteynerdir. Forest daxilində müxtəlif domain ağacları ola bilər.

Organizational Unit (OU)

İstifadəçiləri, qrupları, kompüterləri məntiqi olaraq təşkil etmək üçün istifadə olunur. Siyasətlərin tətbiqini və səlahiyyətləri daha incə səviyyədə təyin etməyə imkan verir.

Schema

AD-dəki obyekt növləri və onların atributlarını təyin edir. Məsələn, "User" obyektinin "Name", "Email", "Password" atributları kimi.

Əsas Active Directory Obyektləri

Obyekt Təyinatı
User Fərdi istifadəçi hesabıdır
Group Eyni hüquqlara malik istifadəçi dəstəsidir
Computer Domenə qoşulmuş kompüterdir
Printer Şəbəkədə paylaşılan printer obyektidir
Shared Folder Fərqli istifadəçilərin giriş edə biləcəyi paylaşılan qovluqdur

Active Directory-də Giriş və Avtorizasiya Prosesi

  • Authentication (Kerberos/NTLM protokolları ilə)
  • Authorization (Group Policy və NTFS icazələri ilə)
  • Access Token yaradılır və istifadəçinin icazələri müəyyənləşdirilir

Group Policy (GPO): Təhlükəsizlik və Konfiqurasiya İdarəetməsi

GPO nədir? Domen daxilindəki istifadəçilər və kompüterlər üçün tətbiq olunan mərkəzləşdirilmiş siyasətlərdir.

GPO vasitəsilə nə etmək olar?

  • Windows konfiqurasiyalarını təyin etmək
  • Skriptlərin icrası
  • Software deployment
  • Təhlükəsizlik siyasətləri (parol uzunluğu, ekran kilidi və s.)

Active Directory Domain Services (AD DS)

AD-nin əsas funksionallığını təmin edir. Domain Controller (DC) adlanan serverlər üzərindən fəaliyyət göstərir. AD DS aşağıdakı funksiyaları təmin edir:

  • Authentication & Authorization
  • LDAP-based directory service
  • Replication services
  • DNS integration

Active Directory və DNS Əlaqəsi

AD düzgün fəaliyyət göstərməsi üçün DNS xidmətinə güvənir. Domenə qoşulma, replikasiya, GPO tətbiqi kimi proseslər DNS olmadan mümkün deyil.

Replikasiya və Fault Tolerance

Multi-Master Replication

Hər bir Domain Controller məlumat bazasının bir nüsxəsini saxlayır və dəyişikliklər bir-birinə sinxronizasiya olunur.

Sites and Services ilə coğrafi zonalar arası replikasiya optimallaşdırılır.

FSMO Roles (Flexible Single Master Operations)

  • Schema Master
  • Domain Naming Master
  • RID Master
  • PDC Emulator
  • Infrastructure Master

Active Directory Təhlükəsizliyi

Hücum Növləri

  • Kerberoasting: Service Account-ların biletlərinin sındırılması
  • Pass-the-Hash / Pass-the-Ticket
  • Golden Ticket / Silver Ticket
  • DCsync Attack (LSA Secret məlumatlarının oğurlanması)
  • Privilege Escalation və Lateral Movement

Təhlükəsizlik Taktikaları

  • Admin Tiering Model (Tier 0, 1, 2 bölməsi)
  • Protected Users Group
  • LAPS (Local Admin Password Solution)
  • Credential Guard & Device Guard
  • Attack Surface Reduction Rules
  • Event Monitoring & SIEM

Azure Active Directory ilə Müasir İnteqrasiya

  • Hybrid Identity: On-premise AD ilə Azure AD inteqrasiyası (Azure AD Connect vasitəsilə)
  • SSO və MFA: Mərkəzləşdirilmiş giriş və çoxfaktorlu identifikasiya
  • Conditional Access: Risk əsaslı giriş nəzarəti
  • Microsoft Entra (Yeni Azure Identity İdarəetməsi)

Real Senarilər: Dövlət Qurumları üçün AD İstifadəsi

Şəbəkə və Giriş Nəzarəti

  • Müxtəlif dövlət təşkilatlarında yüzlərlə istifadəçinin və cihazın idarə edilməsi
  • Tək domen və ya multiple forest strukturunda idarəetmə

Məlumatların Qorunması

  • Fərqli səviyyəli (gizli, məxfi, açıq) informasiyaların ayrılması üçün Group Policy və Access Control Lists tətbiqi
  • Sensitive məlumatlar üçün ayrı OU-lar və GPO-lar

İzlənmə və Hesabatlılıq

  • SIEM ilə logların monitorinqi
  • İstifadəçi fəaliyyətinin analizi (Logon/Logoff hadisələri, GPO dəyişiklikləri və s.)

Active Directory Pentest: Dərin Təhlil və Praktiki Yanaşmalar

Giriş: Active Directory niyə hədəfdir?

Active Directory (AD) — Microsoft tərəfindən hazırlanmış, identifikasiya və autentifikasiya xidmətləri təqdim edən geniş yayılmış mərkəzləşdirilmiş idarəetmə sistemidir. AD mühitləri orta və böyük ölçülü təşkilatlarda istifadəçi hesabları, qruplar, kompyuterlər, printerlər və digər resursları idarə etmək üçün əsas sütundur.

Buna görə də, bir təşkilatın təhlükəsizlik dəyərləndirilməsi zamanı AD sızma testləri (Red Team/Blue Team senariləri daxil olmaqla) mühüm yer tutur. Hücumçular, bu infrastruktur vasitəsilə ilkin giriş əldə etdikdən sonra lateral movement, privilege escalation və data exfiltration kimi mərhələləri reallaşdıra bilirlər.

Active Directory Arxitekturasına Baxış

Əsas Terminlər:

  • Domain: AD mühitində loqik sərhəddir. Məsələn, kerpak.local.
  • Domain Controller (DC): Domain-i idarə edən və autentifikasiya proseslərini həyata keçirən server.
  • Organizational Unit (OU): İdarəetməni asanlaşdırmaq üçün istifadə olunan loqik konteyner.
  • Group Policy Objects (GPO): Konfiqurasiyaların istifadəçilərə və sistemlərə tətbiq olunması.
  • Kerberos & NTLM: Autentifikasiya protokolları.

Active Directory Pentest Fəlsəfəsi və Mərhələləri

Reconnaissance (Məlumat Toplama)

İlk mərhələdə məqsəd — şəbəkə haqqında passiv və aktiv şəkildə məlumat toplamaqdır.

Alətlər:

  • nmap, masscan – port scanning (135, 139, 445, 88, 389, 636, 3268, 9389)
  • crackmapexec, rpcclient, smbclient – DC və SMB servisindən informasiya çıxarılması
  • BloodHound, SharpHound – graph əsaslı nəzarət və zəiflik analizləri
  • ldapsearch, ADRecon, PowerView

Məlumat Toplama Fokusları:

  • Domain adları və strukturu
  • Domain Controllers
  • İstifadəçi və admin qrupları
  • Trust relationships (inter-domain / forest trust)
  • Service Principal Names (SPNs)
  • Group Membershiplər və GPO-lar

İstifadəçi Tapma və Brute Force

Alətlər:

  • kerbrute, GetNPUsers.py, crackmapexec, hydra

Texnikalar:

  • AS-REP Roasting: DONTREQPREAUTH atributu olan istifadəçiləri hədəfləyib, Kerberos hash-larını çıxarmaq.
  • Kerberoasting: SPN-ləri olan istifadəçilərin service ticket-lərini əldə edib, offline parol brute-force.
  • SMB Null Sessions: RPC vasitəsilə açıq informasiya əldə edilməsi.

Local Privilege Escalation və Lateral Movement

Alətlər:

  • mimikatz, rubeus, secretsdump.py, impacket
  • PsExec, WMIexec, SMBexec, SharpRDP

Texnikalar:

  • Pass-the-Hash (PtH)
  • Pass-the-Ticket (PtT)
  • Over-Pass-the-Hash
  • DC Sync Attack – Domain Controller-dən bütün hash-ların çıxarılması
  • Kerberos Ticket Manipulation – Golden Ticket, Silver Ticket
  • Token Impersonation

Persistence və Erişim Qoruma

Texnikalar:

  • GPO dəyişiklikləri ilə script-lərin əlavə olunması
  • Skeleton Key implant
  • AdminSDHolder və SDProp mexanizminin istismarı
  • Scheduled Tasks, Services vasitəsilə qalıcı script-lər

BloodHound ilə AD Dominasiya Analizi

BloodHound, AD-də graph analizləri ilə zəiflikləri və hücum yollarını aşkarlamaq üçün mükəmməl vasitədir.

Əsas Attack Paths:

  • ShortestPathToDomainAdmin
  • AddMember, ForceChangePassword, AllExtendedRights, GenericAll, GenericWrite, WriteDACL
  • ACL Abuse (Access Control Lists)

Müdafiə Perspektivindən: Blue Team-ə Nə Lazımdır?

Təhlükələrin Aşkarlanması üçün:

  • Event ID 4769, 4624, 4672, 4720, 4726, 4740 - monitor edilməlidir
  • Sysmon + SIEM inteqrasiyası (ELK, Splunk)
  • Honeytoken istifadəçi hesabları

Ən Yaxşı Təcrübələr:

  • LAPS (Local Administrator Password Solution)
  • Admin hesablarının ayrılması
  • PowerShell restriksiyaları və audit
  • SMB signing, LDAP signing aktivləşdirilməsi
  • Privilege Access Workstation (PAW) modeli
