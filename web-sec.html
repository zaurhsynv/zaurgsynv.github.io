Web Security

Veb Tətbiqlərin Təhlükəsizlik Strategiyaları və Taktikaları

Giriş: Web Təhlükəsizliyin Əhəmiyyəti

Web tətbiqləri müasir dövrdə istər şəxsi, istərsə də korporativ səviyyədə informasiya mübadiləsinin əsas mərkəzinə çevrilmişdir. Məlumat bazaları, istifadəçi hesabları, maliyyə əməliyyatları və daha çoxu bu tətbiqlər üzərindən həyata keçirilir. Nəticə olaraq, web tətbiqlər hücumçular üçün ən cəlbedici hədəflərdən birinə çevrilir.

Təhlükə Spektri: Ən Yayılan Hücum Növləri

SQL Injection

İnput sahələri düzgün filtrlənmədikdə, hücumçular SQL sorğularını manipulyasiya edərək məlumat bazasına müdaxilə edə bilərlər. Bu, istifadəçi məlumatlarının oğurlanmasına, dəyişdirilməsinə və ya silinməsinə səbəb ola bilər.

admin' OR '1'='1

Cross-Site Scripting (XSS)

Bu hücum növü, istifadəçilərin brauzerində zərərli skriptlərin icra olunmasına imkan verir. XSS sayəsində hücumçu session cookie-ləri oğurlaya və ya istifadəçiyə zərərli linklər göndərə bilər.

Cross-Site Request Forgery (CSRF)

CSRF hücumunda, istifadəçi login olmuş halda zərərli bir sorğunu bilmədən serverə göndərir. Əgər tətbiq CSRF token istifadə etmirsə, bu əməliyyat uğurla həyata keçirilə bilər.

Command Injection

Komanda inyeksiyası zamanı hücumçu serverdə əmrlərin icrasına nail olur. Bu tip hücumlar əsasən eval(), exec(), system() funksiyaları üzərindən həyata keçirilir.

Directory Traversal

Bu metodla hücumçu serverdəki sistem fayllarına çıxış əldə edə bilər. Məsələn:

../../../etc/passwd

OWASP Top 10: Web Təhlükəsizlik Standartları

OWASP (Open Web Application Security Project) təşkilatı tərəfindən yayımlanan "Top 10" siyahısı ən kritik web tətbiq zəifliklərini əhatə edir:

  1. Broken Access Control
  2. Cryptographic Failures
  3. Injection
  4. Insecure Design
  5. Security Misconfiguration
  6. Vulnerable and Outdated Components
  7. Identification and Authentication Failures
  8. Software and Data Integrity Failures
  9. Security Logging and Monitoring Failures
  10. Server-Side Request Forgery (SSRF)

Təhlükəsizlik Mimarisi: Müdafiə Strategiyaları

Input Validation və Sanitization

İstifadəçi daxil etdiyi bütün məlumatlar ciddi şəkildə yoxlanılmalı və zərərli kodların daxil edilməsinin qarşısı alınmalıdır.

Access Control və Role-Based Access (RBAC)

Hər istifadəçiyə yalnız ehtiyac duyduğu səviyyədə icazə verilməlidir. "Least Privilege" prinsipi əsas alınmalıdır.

HTTPS və TLS

Məlumatların şifrələnməsi üçün SSL/TLS sertifikatları istifadə olunmalı və HTTP Strict Transport Security (HSTS) konfiqurasiya edilməlidir.

Security Headers

Müasir brauzerlər üçün təhlükəsizlik başlıqları tətbiq olunmalıdır:

  • Content-Security-Policy (CSP)
  • X-Frame-Options
  • X-XSS-Protection
  • Referrer-Policy

Real Dünyadan Nümunələr

Yahoo (2013–2014)

Bir XSS zəifliyi nəticəsində yüz milyonlarla istifadəçi məlumatı oğurlanmışdı.

Capital One (2019)

AWS misconfig səbəbilə hücumçu S3 bucket-lərə giriş əldə edərək 100 milyondan çox müştəri məlumatını oğurladı.

British Airways (2018)

Magecart hücumu ilə ödəniş məlumatları 380,000-dən çox müştəridən oğurlandı.

Nəticə

Web Security yalnız texniki məsələ deyil – bu, istifadəçi etibarını qorumağın, məlumat təhlükəsizliyini təmin etməyin və hüquqi məsuliyyətdən yayınmağın əsas yoludur. Təhlükəsizlik heç zaman tək bir auditlə yekunlaşmır – bu davamlı prosesdir. Tətbiqlərin həyat dövrünün hər mərhələsində təhlükəsizlik əsas dəyər olaraq qalmalıdır.

Tövsiyələr və Ən Yaxşı Təcrübələr

Secure-by-Design Yanaşması

Web tətbiq hazırlanmasının ilk mərhələsindən etibarən təhlükəsizlik əsas prinsip kimi qəbul olunmalıdır. Kodlaşdırmadan əvvəl təhlükəsizlik riskləri nəzərə alınmalı, dizayn sənədlərində məlumat axınları və hücum səthləri aydın şəkildə qeyd olunmalıdır.

Kod Baxışları və Static Analysis

Kod yazıldıqdan sonra manual və ya alətlərlə (məsələn, SonarQube, Semgrep) kodun təhlükəsizlik baxımından təhlili aparılmalıdır. Bu, zəifliklərin erkən mərhələdə aşkar edilməsinə kömək edir.

Dependency Management və Software Composition Analysis

Üçüncü tərəf kitabxanaları daim yenilənməlidir. OWASP Dependency-Check, Snyk, və ya Dependabot vasitəsilə zəif versiyalar avtomatik təyin olunub yenilənə bilər.

Loglama və Monitoring

Hər bir vacib əməliyyat üçün audit izi saxlanılmalı, anomal davranışlar real-time təhlil olunmalıdır. Brute-force hücumlar, login uğursuzluqları və məlumat axını loglanmalıdır. Bu məlumatlar SIEM sistemlərinə ötürülməlidir.

Bug Bounty və VDP Proqramları

Tətbiqinizi ictimaiyyətə təqdim etdikdən sonra VDP (Vulnerability Disclosure Policy) və ya bug bounty proqramı elan etməklə daha geniş təhlükəsizlik araşdırmalarını təşviq edə bilərsiniz. Platformalar: HackerOne, Bugcrowd, Intigriti.

Web Tətbiqlərin Təhlükəsiz Arxitekturası: Bir Nümunə

Aşağıdakı best-practice arxitektura təhlükəsizlik baxımından gücləndirilmiş web tətbiq dizaynını əks etdirir:

  • Frontend (React/Vue) – Yalnız minimal məlumat saxlayır. LocalStorage yerinə HttpOnly cookie istifadə olunur.
  • Backend (Node.js/Django) – REST API-lər CSRF və rate limiting ilə qorunur.
  • Reverse Proxy (NGINX) – SSL termination, redirect rules və security headers buradan tənzimlənir.
  • WAF (Web Application Firewall) – Hücum pattern-ləri bloklanır. Məsələn: AWS WAF, Cloudflare WAF.
  • Database Layer – Parameterized queries və minimal user permissions.
  • Logging & Monitoring – ELK Stack ilə real-time təhlükə təhlili və alert mexanizmləri.

İxtisaslaşma Yol Xəritəsi: Web Təhlükəsizlik üzrə Karyera

Başlanğıc

  • Github'dakı CVE-lərin analizi

Yüksək səviyyə

  • DevSecOps inteqrasiyası
  • SSRF, Race Condition, Logic Bug təhlili
  • Sertifikatlar (OSWE, eWPTXv3, GWAPT)

Azərbaycan Kontekstində Web Təhlükəsizliyi: Dövlət Qurumları Fokusunda

Azərbaycanın rəqəmsal transformasiya siyasəti çərçivəsində dövlət qurumları sürətlə elektron hökumət (e-Gov) sistemlərinə keçid edir. Portallar, xidmət platformaları, onlayn qeydiyyat sistemləri və vətəndaşla əlaqə kanalları artıq veb əsaslıdır. Bu isə eyni zamanda dövlət sektorunu potensial kiber hücumlar üçün əsas hədəfə çevirir.

Əsas Təhlükələr

Dövlət qurumlarına qarşı hücumlar aşağıdakı məqsədlərlə həyata keçirilə bilər:

  • Məlumat sızması: Vətəndaş məlumat bazalarının (şəxsiyyət vəsiqəsi, pasport, vergi, sağlamlıq) oğurlanması.
  • Xidmətin dayandırılması (DoS/DDoS): Dövlət portallarının fəaliyyətinin iflic edilməsi.
  • Deface və ya imicə zərər: Saytların ön səhifəsinin dəyişdirilərək ictimai panika və ya siyasi mesaj yayılması.
  • Zərərli kodun yerləşdirilməsi: İstifadəçilərin brauzerlərinə yönəlik zərərli skriptlərin inject olunması.

Cari Çətinliklər

  • Resurs çatışmazlığı – Bir çox dövlət qurumunda xüsusi təhlükəsizlik komandası və ya SOC (Security Operations Center) mövcud deyil.
  • Köhnəlmiş texnologiyalar – Legacy sistemlər və zəif təhlükəsizlik praktikaları hələ də geniş yayılıb.
  • Audit boşluqları – Web tətbiqlərinə dair penetrasiya testləri ya ümumiyyətlə aparılmır, ya da formal şəkildə həyata keçirilir.
  • Zəif konfiqurasiya və izləmə – Web serverlərdə təhlükəsizlik başlıqları, WAF, HSTS kimi əsas tədbirlər çox vaxt tətbiq edilmir.

Dövlət üçün Tövsiyələr

  • Milli Web Təhlükəsizlik Standartları – Dövlət İT sistemləri üçün OWASP Top 10 əsasında lokal təhlükəsizlik çərçivəsi hazırlanmalıdır.
  • Mərkəzləşdirilmiş SOC və SIEM – Əsas qurumlar üçün ortaq bir Təhlükəsizlik Əməliyyat Mərkəzi (SOC) yaradılaraq bütün giriş-çıxışların izlənməsi təmin olunmalıdır.
  • Mütəmadi Pentest Proqramları – İllik və ya yarımillik penetration test planları dövlət səviyyəsində tənzimlənməli və nəticələr hesabatlandırılmalıdır.
  • DevSecOps Mədəniyyəti – Dövlətin proqram təminatı komandaları DevSecOps yanaşmasını mənimsəməli və təhlükəsizlik CI/CD axınlarına inteqrasiya olunmalıdır.
  • Təlim və Sertifikatlaşma – Dövlət İT işçiləri üçün web təhlükəsizlik üzrə mütəmadi təlimlər təşkil edilməli, eWPT, GWAPT kimi sertifikatlarla motivasiya artırılmalıdır.

Müsbət Nümunələr və İrəliləyişlər

  • DİA və E-GOV portalının təkmilləşdirilməsi – Son illərdə E-Gov infrastrukturu üzərində görülən işlər nəticəsində daha təhlükəsiz protokollar və identifikasiya metodları tətbiq olunur.
  • AzInTelecom SOC – Dövlət bulud sistemləri üçün qurulan bu mərkəz təhlükələrin analizi və cavablandırılması üçün əhəmiyyətlidir.
